### 整体架构概述（Hierarchical Supervisor Pattern）

- **顶层**：一个 **Supervisor（分发器/协调者）** 负责理解用户整体需求（例如：“帮我规划7天日本东京+京都旅行，预算2万人民币，带小孩”），分解任务，决定调用哪些专业助手，收集结果，合成最终旅行计划，并处理迭代（用户不满意时重新规划）。
- **中层**：多个 **专业助手（Assistants）**，每个助手是一个独立的 **Subgraph（子图）** 每个子助手专注一个领域，内部可包含多个节点（LLM 调用 + 工具调用），打包后直接作为主图的一个节点使用。
- **底层**：工具（Tools）和外部服务 每个助手可共享或独占工具（如搜索航班、查天气等）。

### 推荐的专业助手（Subgraphs）划分

根据旅行规划的典型流程，建议拆分成以下 6-8 个核心助手（可根据需求增减）：

1. **需求收集与偏好助手（Preferences & Input Assistant）**
   - 负责与用户多轮对话，提取/澄清关键信息（出发地、目的地、天数、预算、出行人数、偏好：家庭/美食/冒险/文化等、特殊需求如带小孩/老人）。
   - 输出结构化用户画像。
2. **行程规划助手（Itinerary Planner Assistant）**
   - 核心大脑：基于偏好生成日程框架（Day-by-Day itinerary），分配天数、城市间移动、交通方式大纲。
   - 可迭代优化（用户反馈后调整）。
3. **交通助手（Transportation Assistant）**
   - 查询航班、火车、高铁、租车、自驾等实时/推荐选项，支持多程联运。
   - 考虑预算、时间、转机便利性。
4. **住宿助手（Accommodation Assistant）**
   - 推荐酒店、Airbnb、民宿、度假村等，按位置、预算、评分、设施（家庭房、无障碍等）筛选。
5. **活动与景点助手（Activities & Attractions Assistant）**
   - 推荐当地景点、体验活动、餐厅、文化活动等。
   - 结合天气、季节、用户兴趣过滤，避免过于密集。
6. **天气与实用信息助手（Weather & Practical Info Assistant）**
   - 获取全程天气预报、时差、当地习俗、注意事项（安全、着装、节日等）。
   - 生成打包清单（Packing List）。
7. **预算与优化助手（Budget & Optimization Assistant，可选）**
   - 整体预算拆分、性价比优化、备选方案生成（经济/舒适/豪华包）。
8. **总结与输出助手（Final Synthesizer Assistant，可由 Supervisor 兼任）**
   - 将所有结果整合成美观、结构化的最终计划（PDF/ Markdown 格式），包括地图建议、预订链接等。

### 工作流程（Graph Flow）

1. **入口**：用户输入 → 需求收集助手（或直接进入 Supervisor）。
2. **Supervisor 循环**：
   - 分析当前状态和用户最新反馈。
   - 决定下一步：调用某个/多个助手（支持并行 Send），或结束。
   - 助手执行后返回结果，更新全局状态，再回到 Supervisor 审查。
3. **并行执行**：例如同时调用交通助手 + 住宿助手 + 天气助手，加速规划。
4. **Human-in-the-Loop**：在关键节点（如最终计划生成前）中断，等待用户确认/修改。
5. **结束**：生成完整旅行计划（可导出 PDF、发送邮件），支持后续聊天修改（“把第二天改成海边”）。
